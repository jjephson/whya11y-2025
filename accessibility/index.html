Accessibility Statement

WhyA11y is committed to ensuring digital accessibility for people with disabilities. We are continually improving the user experience for everyone and applying the relevant accessibility standards.

Conformance Status

The WhyA11y website is designed to conform to the Web Content Accessibility Guidelines (WCAG) 2.1 at the AA level. These guidelines explain how to make web content more accessible for people with disabilities and more user-friendly for everyone.

We also strive to meet the requirements of EN 301 549, the European accessibility standard for ICT products and services.

Accessibility Features

Our website includes the following accessibility features:

Navigation and Structure

  • Skip Links: A "Skip to main content" link is available at the top of every page for keyboard users
  • Semantic HTML: Proper heading structure (H1, H2, H3, etc.) and landmark regions
  • Breadcrumbs: Clear navigation path displayed on all pages except the homepage
  • Consistent Navigation: Main navigation menu is consistent across all pages

Visual Design and Colors

  • High Contrast: All text meets WCAG AA contrast requirements (4.5:1 for normal text, 3:1 for large text)
  • Dark/Light Theme: Toggle between dark and light themes to suit your preference
  • Focus Indicators: Clear, visible focus indicators for keyboard navigation
  • Responsive Design: Website adapts to different screen sizes and orientations

Content and Typography

  • Readable Fonts: Atkinson Hyperlegible font family for improved readability
  • Scalable Text: Text can be resized up to 200% without loss of functionality
  • Clear Language: Content is written in clear, simple language
  • Descriptive Links: Link text clearly indicates the destination

Images and Media

  • Alt Text: All meaningful images include descriptive alternative text
  • Decorative Images: Decorative images are marked with empty alt attributes
  • Explicit Dimensions: All images have width and height attributes to prevent layout shifts

Code Examples

  • Syntax Highlighting: Code blocks use high-contrast color schemes
  • Readable Code: Code examples are formatted for readability
  • Accessible Patterns: All code examples demonstrate accessible implementation patterns

Keyboard Navigation

Our website can be navigated entirely using a keyboard:

  • Use Tab to move between interactive elements
  • Use Enter or Space to activate buttons and links
  • Use Escape to close any open dialogs or menus
  • Use Arrow keys to navigate within components

Assistive Technology Compatibility

Our website is compatible with:

  • Screen readers (NVDA, JAWS, VoiceOver, TalkBack)
  • Screen magnification software
  • Voice recognition software
  • Switch navigation devices
  • High contrast mode settings

Known Limitations

While we strive for comprehensive accessibility, we acknowledge that some areas may have limitations. We are actively working to address any identified issues and continuously improve our accessibility standards.

Feedback and Contact

We welcome your feedback on the accessibility of the WhyA11y website. If you experience accessibility barriers or have suggestions for improvement, please contact us:

  • Email: hello@whya11y.com
  • Subject Line: "Accessibility Feedback"

Please include the following information in your message:

  • The specific page or feature you're having trouble with
  • A description of the accessibility barrier you encountered
  • Your preferred method of contact for follow-up

We aim to respond to accessibility feedback within 2 business days.

Continuous Improvement

Accessibility is an ongoing commitment. We regularly review and update our website to ensure it remains accessible and compliant with current standards. Our development process includes:

  • Regular accessibility audits and testing
  • User feedback integration
  • Staff training on accessibility best practices
  • Monitoring of accessibility standards and guidelines
